# Перформанс-бюджеты и как мерить

Фиксируем пороги (§10/§13). Контент статичен (SSG); тяжёлая интерактивность — ленивые
острова. Числа initial JS — `raw` (несжатые); по сети gzip ≈ ÷3.

## Целевые Core Web Vitals («good»)
- **LCP** < 2.5 s · **INP** < 200 ms · **CLS** < 0.1
- **Lighthouse Performance** ≥ 90 (mobile) / ≥ 95 (desktop)

## Бюджет initial JS (факт на момент Шага 6.2)
Замер — сумма `<script src=/_next/static/...>` в пререндеренном `.next/server/app/*.html`.

| Маршрут | chunks | raw | ≈ gzip | потолок (raw) |
|---|---|---|---|---|
| `/` (хаб, Витрина) | 16 | 913 KB | ~300 KB | ≤ 950 KB |
| `/contacts` | 15 | 882 KB | ~290 KB | ≤ 950 KB |
| `/workshop` | 16 | 924 KB | ~305 KB | ≤ 950 KB |
| `/read` (индекс Читальни) | 10 | 622 KB | ~205 KB | ≤ 700 KB |
| `/read/[book]/[story]` | 11 | 668 KB | ~220 KB | ≤ 700 KB |

Витрина тяжелее Читальни намеренно: Lenis + GSAP + Motion (моушн-острова). Читальня —
без них (нативный скролл, §6). Если маршрут превышает потолок — искать новый тяжёлый
импорт, не попавший в ленивый остров.

## Что держит бюджет (проверяемо грепом по `.next`)
- **`three` (863 KB)** — только ленивый чанк `GrainCanvas` (`ssr:false`); НЕТ в initial
  ни одной страницы (маркер `WebGLRenderer` отсутствует в `*.html`).
- **`d3-force` + граф** — ленивый `WheelGraph` (`ssr:false`), только десктоп.
- **`SplitText`/`GlitchText`** — ленивый `StoryOpening` (грузится при первом зачине, не
  на каждой странице Витрины). Маркер отсутствует в initial-HTML Витрины.
- **Шрифты** — сабсет cyrillic+latin, `display:swap`, preload прозы/системы, авто-fallback (CLS).
- **Картинки** — `next/image` AVIF/WebP (`next.config`), размеры из манифеста (без CLS),
  `priority` для LCP-плашки; `deviceSizes`/`imageSizes` сужены под плашки ≤ 24rem.
- **React Compiler** — `reactCompiler: true`: автомемоизация островов (INP), особенно «Колесо».

## Как измерить (нужен браузер — headless здесь ненадёжен)
1. `npm run build && npm run start` → http://localhost:3000
2. **Lighthouse:** DevTools → вкладка Lighthouse → Mobile → Analyze; либо
   `npx lighthouse http://localhost:3000 --preset=desktop --view`.
3. **CWV «в поле»:** DevTools → Performance (запись прокрутки/кликов) или расширение Web Vitals.
4. Прогнать ключевые: `/`, `/read/<книга>/<рассказ>`, `/workshop`, `/contacts`
   (и на мобильной ширине ~360 px).
5. **Re-замер initial JS** после правок: сумма `/_next` скриптов в `.next/server/app/*.html`.

## Возможные дальнейшие шаги (если бюджет Витрины надо ужать)
- Контактам/Мастерской Lenis+GSAP не нужны по содержанию — можно вынести моушн-обвязку
  из общего `(vitrina)/layout` в под-layout только для хаба/«Колеса». (Заметная переделка —
  пока не делаем: укладываемся в потолок.)
