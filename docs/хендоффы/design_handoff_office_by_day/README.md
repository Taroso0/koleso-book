# Handoff: Уплотнение светлой Витрины — «Офис днём» (Автор + Мастерская)

## Overview
Пересборка **светлых секций Витрины** — «Автор» и «Мастерская» — из «карточки в облаке» в **документ/архив днём**. Смысл: светлые секции — это «офис днём», плотный скучно-достоверный мир документов; именно на его фоне работает «чудо на краю взгляда» из ночных секций. В пустой центральной колонке чудо превращается в декор, поэтому плотность даётся **фактурой и вёрсткой**, а не добавленным текстом.

Соответствует задаче №3 плана ночной полировки. Целевые файлы в кодовой базе:
- `components/vitrina/*` (секция «Автор» живёт рядом с `BooksSection.tsx` / `HeroZachin.tsx` на главной Витрины `app/(vitrina)/page.tsx`)
- `components/workshop/WorkshopCard.tsx` + `app/(vitrina)/workshop/page.tsx` (Мастерская)

## About the Design Files
Файл `Шаг 8 - Офис днём.html` в этом бандле — **дизайн-референс, сделанный в HTML**: прототип, показывающий задуманный вид и поведение, а не продакшн-код для копирования. Задача — **воссоздать этот дизайн в существующей среде проекта** (Next.js App Router + React + TypeScript, стили в `app/globals.css` и токенах `ds-bundle/tokens`), используя устоявшиеся паттерны проекта (server components Витрины, `components/vitrina`, `components/workshop`). React/Babel-обвязка и панель Tweaks в HTML — это только инструмент прототипа, в продакшн они не переносятся.

## Fidelity
**High-fidelity.** Финальные цвета (oklch), типографика, отступы, наклоны и состояния заданы точно — воссоздавать пиксель-в-пиксель средствами кодовой базы. Панель Tweaks в прототипе перечисляет варианты фактуры; в продакшне закрепить **дефолтные** значения (см. ниже), tweak-контролы не переносить.

## Screens / Views

Обе секции делят единый макет **«рабочее поле»**: двухколоночная сетка `[поле-слева 128px] [контент 1fr]` с вертикальной линейкой-разделителем между колонками. Левая колонка — служебные пометки на полях (моно). Секции обёрнуты в колонтитулы документа (верхний/нижний run-head/run-foot).

### 1. Колонтитулы документа (обрамление)
- **Верхний (`.runhead`)**: flex, space-between, `border-bottom: 1px solid var(--hairline)`, padding `10px 0`. Слева моно «боковым зрением · служебные страницы», справа «лист 04–05».
- **Нижний (`.runfoot`)**: `border-top`, `margin-top: 72px`. Слева «littera scripta manet — написанное слово остаётся», справа «продолжение — ночью».
- Оба текста — `.mono-mark` (см. токены).

### 2. Секция «Автор» — личный лист
- **Purpose**: смысловое позиционирование голосом автора вместо карточки-цитаты.
- **Layout**: рабочее поле. Левое поле: моно-пометки «разд. 04», короткая линейка (22×1px), «автор», «рег. 14.02.2024». Контент: заголовок `<h2>Автор</h2>`, под ним `.h-rule` (48×1px, `var(--ink)` при opacity 0.4), затем «личный лист» — `article.doc`.
- **`article.doc`** (машинописный лист):
  - `background: var(--sheet)` (по умолчанию) или `var(--manila)` при tone=Манила; `border: 1px solid var(--hairline)`; тень `0 14px 28px -22px oklch(0.2 0.02 250 / 0.45)`; `margin-top: 32px`; `position: relative`.
  - **Шапка `.doc-head`**: flex space-between, padding `14px 26px`, `border-bottom: 1px solid var(--hairline)`. Слева «личный лист · экз. единственный», справа «форма № 2-А» (обе `.mono-mark`).
  - **Тело `.doc-body`**: grid `1fr 180px`, gap `clamp(24px,4vw,44px)`, padding `26px 26px 24px`.
    - Левая колонка: `.mono-mark` lead-in «кирилов, евгений · род занятий: автор · жанр: офисная готика»; затем `.voice` — антиква, `font-size: 1.08rem`, `line-height: 1.7`, `max-width: 46ch`, цвет `var(--ink)`. **Текст (точный):** «Евгений Кирилов пишет „офисную готику“ — чудо, спрятанное в сером корпоративном быту: в опенспейсе, в фонаре в три часа ночи, в системном уведомлении. Тепло, иронично, по-человечески — а не торжественно.»
    - Строка подписи `.sig-row`: flex, baseline, gap 14px, margin-top 26px. `.sig` — курсив 1.05rem, `border-bottom: 1px dotted var(--hairline)`, padding `0 18px 2px 4px`, текст «Е. Кирилов»; рядом `.mono-mark` «подпись».
    - Правая колонка `figure`: `.photo` — плейсхолдер фотокарточки, `aspect-ratio: 3/4`, `border: 1px solid var(--hairline)`, фон — диагональная штриховка `repeating-linear-gradient(45deg, transparent 0–7px, var(--hairline-soft) 7–8px)`. Два уголка `.corner.tl` / `.corner.br` (14×14px, L-образные рамки из hairline, вынесены на −5px). Подпись `.photo-cap` (`.mono-mark`, центр) — **«место для фотокарточки · скоро»**. Метка «скоро» относится ТОЛЬКО к фото, не к автору.
  - **Штамп `.stamp`** (единственный натриевый акцент секции): `position: absolute; left: 26px; bottom: -18px`; `transform: rotate(var(--stamp-tilt))` (дефолт −5°); `border: 2px solid var(--sodium)`; `color: var(--sodium)`; `background: var(--paper-bg)`; opacity 0.92; padding `7px 12px 6px`; моно 9px, letter-spacing 0.18em, uppercase. Содержимое: жирная строка «принято к прочтению» + «вх. № 2026-071».

### 3. Секция «Мастерская» — стол с черновиками
- **Purpose**: building-in-public для прозы — третья книга как физические листы разного тона.
- **Layout**: рабочее поле. Левое поле: «разд. 05», линейка, «мастерская», «обн. 12.07.2026». Контент: `<h2>Мастерская</h2>`, `.h-rule`, лид `.lede` (1.02rem, line-height 1.7, `var(--ink-soft)`, max-width 52ch): «Третья книга растёт на глазах: фрагменты, черновики, заметки и новые иллюстрации. Building-in-public для художественной прозы.»
- **Стол `.desk`**: grid `1.2fr 1fr 0.9fr`, gap `clamp(18px,3vw,30px)`, align-items start, margin-top 40px. Три листа `.draft` (border hairline, тень как у `.doc`, padding `16px 18px 18px`), каждый со своим наклоном и тоном:
  - **d1** — `rotate(calc(var(--tilt) * -0.6))`, фон `--sheet`. Моно-тег «черновик · фрагмент 07 · 12.07». Фрагмент `.frag` (курсив, 0.98rem, `--ink-soft`): «Лифт остановился между этажами — ~~и погас свет~~ и кто-то вежливо кашлянул.» (`.strike` = line-through цветом `oklch(0.235 0.012 250 / 0.35)`). Ниже `.lines` — блок 54px с горизонтальной линовкой (repeating-linear-gradient 18px шаг, hairline).
  - **d2** — `rotate(calc(var(--tilt) * 0.8)) translateY(14px)`, фон `var(--manila)`. Тег «заметка · без даты». Фрагмент: «Проверить: как звучит уведомление, если его никто не читает.» + `.lines`.
  - **d3** — `rotate(calc(var(--tilt) * -0.3)) translateY(4px)`, фон `--sheet`. Тег «иллюстрация · тушь · скоро». `.ill` — плейсхолдер `aspect-ratio: 4/5` с той же диагональной штриховкой, что у `.photo`.
- **Ссылка `.read`**: `Зайти в Мастерскую →`, шрифт sans 0.95rem, `border-bottom: 1px solid var(--sodium)`, padding-bottom 3px, hover → цвет `--sodium`. Единственный натрий этой секции.

## Interactions & Behavior
- Статичный контент; интерактив ограничен hover-состояниями. Ссылки: default `color: var(--ink)`, hover `var(--sodium)`.
- **Ссылка «Зайти в Мастерскую»** ведёт на `/workshop` (`app/(vitrina)/workshop/page.tsx`).
- Наклоны листов и штампа — статические `transform: rotate(...)`, без анимации.
- **Responsive** (≤760px): рабочее поле схлопывается в одну колонку — левое поле становится горизонтальной полосой сверху (`border-bottom` вместо `border-right`, пометки в ряд, короткие линейки скрыты). `.doc-body` → одна колонка, `.photo` ограничивается `max-width: 200px`. `.desk` → одна колонка, наклоны d2/d3 упрощаются до `rotate(calc(var(--tilt)*0.4))`.

## State Management
Состояния приложения не требуется — это статические server components Витрины. В прототипе `data-*` атрибуты на `<body>` + панель Tweaks управляют фактурой; в продакшне это **фиксированные дефолты**, не переключатели:
- `data-marks="on"` — пометки на полях видны.
- `data-ruled="on"` — тонкая линовка фона контентной колонки: `repeating-linear-gradient(180deg, transparent 0–27px, var(--hairline-soft) 27–28px)`.
- `data-stamp="on"` — штамп виден.
- `data-doc-tone="white"` — личный лист на белом (`--sheet`). Альтернатива «манила» — не дефолт.
- `--stamp-tilt: -5deg`, `--tilt: 1.5deg`.

## Design Tokens

Цвета (oklch — совпадают с направлением палитры «Документ»):
- `--paper-bg: oklch(0.968 0.006 160)` — фон страницы (институциональный, насыщенность НЕ повышать).
- `--sheet: oklch(0.988 0.003 95)` — бумага листа.
- `--manila: oklch(0.915 0.028 92)` — тон «манила» (второй тон бумаги).
- `--manila-deep: oklch(0.86 0.035 90)` — резерв.
- `--ink: oklch(0.235 0.012 250)` — основной текст.
- `--ink-soft: oklch(0.47 0.012 250)` — вторичный текст.
- `--faint: oklch(0.6 0.01 250)` — моно-пометки.
- `--sodium: oklch(0.7 0.14 78)` — натриевый акцент. **Правило: максимум один натриевый акцент на экран/секцию** (штамп у Автора, подчёркивание ссылки у Мастерской).
- `--hairline: oklch(0.235 0.012 250 / 0.14)` — рамки/разделители.
- `--hairline-soft: oklch(0.235 0.012 250 / 0.07)` — штриховки/линовка.

Типографика:
- Serif (антиква): `"Source Serif 4", Georgia, serif` — тело, голос автора, подпись.
- Sans: `Inter, system-ui` — только ссылка `.read`.
- Mono: `"JetBrains Mono", ui-monospace, monospace` — все служебные пометки.
- `.mono-mark`: 9.5px, letter-spacing 0.14em, uppercase, `var(--faint)`.

Отступы/размеры: контейнер max-width 960px, padding `40px 24px 96px`. Поле-колонка 128px. Тени листов: `0 12–14px 24–28px -20/-22px oklch(0.2 0.02 250 / 0.45)`.

## Нельзя (ограничения задачи)
- Добавлять контент ради заполнения — плотность даётся фактурой и вёрсткой, не текстом.
- Градиентные фоны (штриховки/линовка — не декоративные градиенты, а бюрократическая фактура).
- Поднимать насыщенность фона выше институциональной (`--paper-bg` не трогать в сторону яркости).
- Больше одного натриевого акцента на секцию.
- Возвращать метку «СКОРО» как статус автора — она относится только к плейсхолдеру фото/иллюстрации.
