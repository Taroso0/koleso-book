# Архитектурные решения — почему так устроено

Живой справочник инженерных решений и причин за ними. Одно решение — несколько строк:
**что решили · почему · где это в коде**.

**Чем этот файл отличается от соседей.** `концепция.md` — источник истины по смыслу и
стилю (ЧТО и ЗАЧЕМ мы строим, решает автор). Этот файл — КАК и ПОЧЕМУ именно так
(решает инженер). `задачи.md` — что ещё не сделано. `архив/спецификация-постройки.md` —
исходный план стройки, документ 2026 года, читать только ради истории.

**Правило ведения.** Решение попадает сюда, когда принято, и правится, когда меняется.
Отменённое решение не удаляется — переезжает в «Отвергнутое» с причиной. Если код
разошёлся с записью здесь — прав код, а запись чинится тем же коммитом.

---

## 1. Природа проекта

**Статический сайт (полная SSG), без бэкенда и БД.**
Контент — 2 изданные книги плюс медленно растущая 3-я; он известен на сборке целиком.
Бэкенд добавил бы точку отказа и стоимость, не дав ничего взамен.
→ `dynamicParams = false` на всех динамических маршрутах; 44 страницы пререндерены.

**ISR не используется.**
Заявлялся в исходной спеке, но не понадобился: обновление контента = новая сборка.
Ни одного `export const revalidate` в проекте.

**Тяжёлая интерактивность — «острова», а не общий каркас.**
WebGL-зерно и граф «Колеса» грузятся через `next/dynamic` с `ssr:false`. Иначе три
сотни килобайт JS ехали бы к читателю, которому нужен только текст.
→ `components/haunted/GrainOverlay.tsx`, `components/wheel/WheelGraph.tsx`

## 2. Две среды — главное правило проекта

**«Витрина» и «Читальня» разведены route-группами и живут по разным законам.**
Инерционный скролл (Lenis) прекрасен на лендинге и невыносим на длинном тексте: глаз
теряет строку. Поэтому это не настройка, а граница в файловой структуре.

| | `(vitrina)` — `/`, `/workshop`, `/contacts` | `(chitalnya)` — `/read/**` |
|---|---|---|
| Скролл | Lenis + GSAP ScrollTrigger | нативный |
| Слой «готики» | зерно, курсор, зачин | нет |
| Носитель тем | — | `.reading-root` (светлая/сепия/тёмная) |

→ `app/(vitrina)/layout.tsx`, `app/(chitalnya)/layout.tsx`

**Lenis ИЛИ ScrollSmoother — никогда оба.** Два скролл-движка дерутся за события:
джанк и сломанные якоря. Связка с GSAP — через `lenis.on("scroll", ScrollTrigger.update)`.
→ `components/motion/SmoothScroll.tsx:20-25`

**Тема чтения ставится до первой отрисовки.** Иначе при F5 в тёмной теме мигает белым.
Пре-пейнт-скрипт в корневом layout читает localStorage и ставит атрибут на `<html>`.
→ `app/layout.tsx` (`READING_INIT`), ключи `kirilov:reading-theme` / `-mode`

## 3. Контент

**Файловый контент + gray-matter + zod. Contentlayer НЕ используем** — проект заброшен.
Валидация на сборке: невалидный frontmatter роняет `next build` с именем файла и поля.
→ `content/schema.ts` (все схемы и канонические id), `lib/content.ts`

**⚠️ MDX не компилируется, хотя расширение `.mdx`.**
Ни `@next/mdx`, ни `@mdx-js/*` не установлены. Тело рассказа — обычная строка, её
рендерит собственный минимальный парсер: **абзацы по пустой строке, `*курсив*` и `***`
как разделитель сцен — и всё**. Заголовок, список, ссылка, JSX отрендерятся буквальным
текстом. Расширение — исторический артефакт; решение «переименовать или подключить MDX»
открыто в `задачи.md`.
→ `components/reader/ProseBody.tsx`

**Контент течёт в одну сторону, ручная вычитка — посередине.**
```
source/*.pdf → ingest_pdf.py → content/_raw/ → [вычитка человеком] →
build_mdx.py → content/stories/*.mdx + плашки → загрузчики → SSG
```
`content/_raw/` — единственное место, где живёт вычитанный текст. Поэтому
`ingest_pdf.py` без `--force` существующие файлы не трогает, а `build_mdx.py` бережёт
ручные `themes[]` и галочки чек-листа.

**«Мастерская» изолирована как единственный шов под будущую CMS.**
3-я книга растёт во времени, и автор может не захотеть править Git. Чтобы отдать
раздел в Payload/Sanity, достаточно переписать тело двух функций — канон не трогается.
→ `lib/workshop.ts`

**Русская микротипографика применяется на сборке, не в рантайме.**
«Ёлочки», неразрывные пробелы, тире считаются один раз при SSG.
→ `lib/typograf.ts`

**Загрузчики читают диск один раз за сборку.** Раньше `getStory` был
`getAllStories().find(...)` — ~144 полных скана корпуса за сборку. Кэш выключен в dev и
тестах, иначе правка контента не подхватывалась бы. Отдают копию массива: вызывающие
сортируют результат на месте, а `Array.sort` мутирует.
→ `lib/buildCache.ts`

**Связность контента проверяется тестами, а не надеждой.** `books/*.json` пишется
руками; страница на неизвестный slug делает `if (!story) return null` — рассказ молча
исчезает. 14 проверок ловят это на CI.
→ `tests/content/integrity.test.ts`

## 4. «Колесо»

**Доступный индекс — канон навигации, граф — улучшение поверх.**
Пространственный граф принципиально недоступен с клавиатуры и скринридера. Поэтому
сначала был написан семантический список «тема → рассказы» (серверный компонент,
нативные `<details>`, работает без JS), и только потом — визуал. На мобильном индекс —
основной и единственный вид.
→ `components/wheel/WheelIndex.tsx`

**Логика графа — чистые функции без DOM.** Их можно считать на сборке и тестировать.
→ `lib/graph.ts` (`buildGraph`/`groupByTheme`/`themeDegree`/`reweight`)

**Укладка предрассчитана на сборке, seed детерминирован.** Иначе узлы «прыгают» при
каждой загрузке и тормозят на слабых машинах. Клиенту приходит готовый массив позиций.
→ `lib/wheelLayout.ts` (LCG-seed, 400 итераций)

**Память «Колеса»: прочитанное стойко перестраивает карту.** Источник правды —
прогресс «Читальни» в localStorage; «Колесо» его только читает.
→ `lib/graph.ts` (`reweightRead`), `components/wheel/readStories.ts`

## 5. Движение и доступность

**Единый шлюз движения.** `reduced = ОС prefers-reduced-motion ИЛИ ручной тумблер`.
Каждый эффект обязан спрашивать этот хук — никаких локальных проверок.
→ `components/motion/MotionProvider.tsx`, `useReducedMotionSafe.ts`

**Паритет, а не «выключено».** У каждого эффекта есть полноценный статический сценарий.
Образец: анимированное WebGL-зерно заменяется статичным SVG-зерном — фактура остаётся
всегда, гейтится только движение.
→ `components/haunted/StaticGrain.tsx`

**Атмосфера дополнительно гейтится по устройству.** Тач, слабое железо, save-data →
статичная деградация: WebGL съедает батарею мобильных.
→ `components/haunted/useHauntedCapability.ts`

**Движется только «сверхъестественное».** Контент стабилен; никакого тотального
parallax. Не больше 3 вспышек в секунду (WCAG 2.3.1), контраст — WCAG AA.

## 6. Стиль и типографика

**Tailwind v4 CSS-first, токены в OKLCH.** OKLCH даёт предсказуемый контраст и
корректную интерполяцию при анимации цвета. Семантические токены готики: `--sodium`
(натриевый свет), `--paper`, `--concrete`, `--monitor`.
→ `app/globals.css` (`@theme inline` поверх shadcn-переменных)

**Дуальная типографика «душа против системы».** Проза — Source Serif 4; интерфейс —
Inter и JetBrains Mono. **Кириллица обязательна во всех шрифтах**, сабсет
`cyrillic+latin`: многие display-гарнитуры её просто не содержат.
→ `lib/fonts.ts`

**Tailwind не сканирует `.design-sync/`.** Каталог закоммичен, и утилита, упомянутая в
бэктиках внутри заметок, становилась мёртвым правилом в прод-CSS. Петля срабатывала.
→ `app/globals.css` (`@source not`)

## 7. Производительность

**Бюджет initial JS — единственный числовой порог по скорости, и он автоматический.**
Метод: сумма уникальных `<script src="/_next/...">` в пререндеренном HTML, несжатые
байты (по сети gzip ≈ ÷3). Гоняется в CI после сборки.
→ `scripts/check-bundle-size.mjs`, `npm run check:size`

Пороги — **храповик**: стоят на ~2,5 % выше факта. Задача — не дать вырасти незаметно,
а не заставить похудеть. Снижаете вес — опускаете порог тем же коммитом; растите
осознанно — поднимаете и объясняете в сообщении коммита.

| Маршрут | факт 2026-07-21 | потолок |
|---|---|---|
| `/` | 960 КБ | 985 КБ |
| `/contacts` | 952 КБ | 975 КБ |
| `/workshop`, `/workshop/[entry]` | 968 / 966 КБ | 990 КБ |
| `/read` | 811 КБ | 835 КБ |
| `/read/[book]` | 808 КБ | 835 КБ |
| `/read/[book]/[story]` | 830 КБ | 855 КБ |

*«Витрина» тяжелее «Читальни» намеренно: Lenis + GSAP + моушн-острова. История чисел:
на Шаге 6.2 было 913 / 668 КБ; +150 КБ дал рост framework-слоя (Next 16.2, React 19.2),
+28 КБ на всех маршрутах — добавленный экран ошибки (`ErrorScene` + `GlitchText`).
Долгосрочная цель — вернуть «Читальню» под 700 КБ; самый крупный резерв в «Витрине» —
заменить three.js + R3F + drei (чанк 864 КБ) на сырой WebGL в `GrainCanvas`.*

**Целевые Core Web Vitals:** LCP < 2,5 с · INP < 200 мс · CLS < 0,1 ·
Lighthouse ≥ 90 mobile / ≥ 95 desktop. Автоматически **не** измеряются — ручной прогон:
`npm run build && npm start`, затем DevTools → Lighthouse (Mobile) или
`npx lighthouse http://localhost:3000 --preset=desktop --view`. Проверять `/`,
`/read/<книга>/<рассказ>`, `/workshop`, `/contacts`, в том числе на ширине ~360 px.

**Что держит бюджет** (проверяется грепом по `.next`):
- `three` — только в ленивом чанке `GrainCanvas`, в initial нет ни одной страницы;
- `d3-force` и граф — ленивый `WheelGraph`, только десктоп;
- `SplitText`/`GlitchText` — ленивый `StoryOpening`, грузится при первом зачине;
- картинки — `next/image` AVIF/WebP, размеры из манифеста (без CLS), `deviceSizes`
  сужены под реальный размер плашек ≤ 24rem;
- React Compiler (`reactCompiler: true`) — автомемоизация островов, прежде всего «Колеса».

## 8. Отвергнутое (чтобы не возвращаться)

- **Contentlayer** — проект заброшен. Вместо него gray-matter + zod.
- **ScrollSmoother вместе с Lenis** — конфликт скролл-движков.
- **ISR** — контент известен на сборке, ревалидировать нечего.
- **`generateSitemaps` (шардинг)** — нужен от 50 000 URL, у нас ~45.
  → `app/sitemap.ts`
- **Web Worker для симуляции графа** — заменён предрасчётом на сборке. Остаток:
  на hover симуляция всё ещё считается в главном потоке (открыто в `задачи.md`).
- **Отдельный UI сносок** — маркер-звёздочка и курсивная расшифровка в конце рассказа
  оставлены как есть. Решение автора.
- **Ротация «День» в покое «Колеса»** (2026-07-10) — тизер на hero печётся на сборке и
  обещает тему максимальной степени; ротация рассинхронизировала бы тизер с картой.
  В покое горит макс-тема («Максимум»).
- **«Мастерская» в переключателе книг «Читальни»** — это раздел «Витрины», а не книга;
  разные среды. Возможность чипа-ссылки зафиксирована в `задачи.md`.
