## 0. Как пользоваться этим документом

Документ предназначен для исполнителя — в первую очередь для **Claude Code**. Он описывает не «что построить» (это задаёт концепция v2.0 «Боковым зрением»), а **как построить**: конкретный стек, структуру репозитория, модель данных, контракты компонентов и карту рисков.

- **Читать в паре с концепцией.** Этот документ — инженерный слой; смысловые решения (офисная готика, Колесо, типографика-герой) берутся из концепции и здесь не повторяются.
- **Версии — ориентир, не догма.** Версии указаны на июнь 2026. Перед установкой свериться: `npm view <pkg> version`. Ставить актуальный stable.
- **Ограничения = страховочные рельсы.** Разделы про доступность, reduced-motion и производительность — не «по возможности», а часть определения готовности (раздел 13).
- **Главный архитектурный факт.** Сайт контентно тяжёлый, но контент статичен (2 изданные книги + медленно растущая 3-я). Это статический сайт (SSG) с островами насыщенной клиентской интерактивности, а не приложение с бэкендом и БД.

## 1. Природа проекта и ключевые архитектурные решения

**Тип.** Статический контент-сайт (Next.js App Router, SSG/ISR) с «островами» тяжёлой интерактивности: Колесо, моушн, haunted-UI. Бэкенд и БД для канона не нужны.

**Два пространства — две среды рендера.** Из концепции: «Витрина» (опыт) и «Читальня» (чтение) живут в разных route-группах App Router со своими layout'ами. Это техническая, а не только смысловая граница: «Витрина» подключает плавный скролл и моушн, «Читальня» — нет (длинный текст несовместим с инерционным скроллом).

**Маппинг трёх опор концепции на технологии.**

- **Офисная готика / haunted UI →** слой эффектов: кастомный курсор, «неправильный» скроллбар, глитч-типографика, WebGL-зерно/CRT-оверлей. Движется только сверхъестественное (раздел 7).
- **Колесо →** контент как граф (узлы-понятия + узлы-рассказы), силовая укладка d3-force, рендер в SVG, перестройка при чтении; обязательный доступный индекс-двойник (раздел 5).
- **Типографика-герой →** дуальная шрифтовая система с кириллицей, вариативные шрифты, кинетика на зачинах (SplitText), токены в OKLCH (раздел 9).


**Важные нюансы стека.**

- **GSAP теперь свободен целиком (с апреля 2025, v3.13+):** все ранее платные плагины (SplitText, ScrollTrigger, ScrollSmoother, MorphSVG, DrawSVG) бесплатны, включая коммерческое использование. Лицензия free-to-use, не open-source: нельзя строить на нём конкурирующий визуальный аниматор — для нашего проекта это неважно.
- **Tailwind v4 — конфиг в CSS, не в JS:** дизайн-токены задаются через `@theme` в `globals.css` в палитре OKLCH. Это удобно для анимации цвета и для дуальной типографики.
- **Motion ≠ старый импорт:** пакет называется `motion`, импорт из `motion/react` (не `framer-motion`).
- **Lenis или ScrollSmoother — выбрать одно.** Lenis легче и дружит с Motion; ScrollSmoother (теперь бесплатный) — для полностью GSAP-пайплайна. Два смус-движка одновременно ставить нельзя (раздел 11).
- **Contentlayer не использовать —** проект заброшен. Контент собирать через MDX + gray-matter + zod (или next-mdx-remote).

## 4. Модель контента и пайплайн из PDF

**Схема контента.** Единый источник правды для Читальни и Колеса — типизированные рассказы с frontmatter, валидируемые zod при сборке.

```ts
// content/schema.ts
type ThemeId =
  | 'soul' | 'light' | 'time' | 'memory' | 'death'
  | 'choice' | 'illusion' | 'fate' | 'human' | 'rebirth';

interface Story {
  slug: string;          // стабильный (дип-линки из Колеса)
  title: string;
  book: 'kolizey' | 'koleso' | 'kniga3';
  order: number;         // позиция в книге
  themes: ThemeId[];     // связи с понятиями Колеса
  firstLine: string;     // «первая строка как событие»
  illustration?: string; // путь к плашке Кучеренко
  // тело рассказа — MDX рядом с frontmatter
}

interface Theme { id: ThemeId; label: string; }   // { id:'rebirth', label:'Перерождение' }
interface Book  { id: string; title: string; year: number; stories: string[]; }
```

**Пайплайн из PDF** (это отдельная работа, см. раздел 11-A1).

- **Извлечение текста.** Скрипт `scripts/ingest-pdf.ts` вытягивает текст постранично (pdftotext / pdf.js), затем чистит: убирает колонтитулы и номера страниц, чинит переносы, нормализует «», тире, неразрывные пробелы, ударения (во́ду).
- **Границы рассказов —** по оглавлению (в обеих книгах есть «Содержание» с порядком); разбиение на 17 + 17 рассказов; маркеры сцен `***` сохранять как разрывы.
- **Микротипографика —** прогон через `typograf` (русские правила) на этапе сборки, а не в рантайме.
- **Иллюстрации —** извлечь из PDF в исходном разрешении отдельно (pdfimages); сохранить в `public/illustrations` и привязать к рассказам через frontmatter.
- **Проверка глазами обязательна.** Печатные PDF/X дают артефакты извлечения; каждый рассказ вычитывается вручную после автоконвертации.

## 5. «Колесо»: модель данных, граф и алгоритм

**Структура.** Граф: 10 узлов-понятий + 34 узла-рассказа; рёбра — членство «рассказ ↔ тема». Около 44 узлов — нагрузка лёгкая, рендер в SVG.

```ts
// lib/graph.ts
function buildGraph(stories, themes) {
  const nodes = [
    ...themes.map(t  => ({ id: t.id,   kind: 'theme', label: t.label })),
    ...stories.map(s => ({ id: s.slug, kind: 'story', label: s.title })),
  ];
  const links = stories.flatMap(s =>
    s.themes.map(themeId => ({ source: s.slug, target: themeId, weight: 1 }))
  );
  return { nodes, links };
}

// Перестройка при чтении: усилить рёбра активного рассказа и его тем,
// пересчитать укладку ДЕТЕРМИНИРОВАННО (фикс. seed), анимировать переходы.
function reweight(graph, activeSlug) { /* ↑weight соседних рёбер, re-run d3-force */ }
```

**Рендер и поведение.**

- **Укладка —** d3-force (радиально/силово), детерминированный seed, ограниченное число итераций; тяжёлую симуляцию выносить в Web Worker.
- **Отрисовка —** SVG-узлы и рёбра; переходы позиций при перестройке — через Motion (layout) или GSAP с плавным easing, чтобы узлы не «прыгали».
- **Перестройка при чтении —** усиление связей активного рассказа; под reduced-motion перестройку НЕ анимировать — только подсветка.
- **Доступный двойник обязателен —** `<WheelIndex />`: семантический список «тема → рассказы», это и есть каноническая навигация; визуальный граф — улучшение поверх неё (раздел 11-B1).

## 6. «Читальня»: приложение для чтения

**Маршруты.** `read/ → read/[book]/ → read/[book]/[story]`. Отдельный layout без Lenis: длинный текст читается нативным скроллом.

- **Reader —** типографика-first: широкие поля, спокойный «документный» фон с высоким контрастом, иллюстрации Кучеренко как editorial-плашки (не частицы).
- **Пагинация —** постраничный режим (CSS columns или вычисляемая разбивка) либо непрерывное чтение; переключатель книг (BookSwitcher).
- **Прогресс чтения —** localStorage/IndexedDB (это боевой сайт, не артефакт — браузерное хранилище доступно): последняя позиция, продолжить чтение.
- **Темы чтения —** светлая/сепия/тёмная для читаемости; атмосфера «офисной готики» — в «Витрине», в «Читальне» приоритет — комфорт глаз.
- **Переход из Колеса —** узел-рассказ ведёт в Читальню; на Next 16 уместен View Transitions API для бесшовного перехода (узел «превращается» в страницу).

## 7. Система движения и «haunted UI»

**Принцип в коде.** Покой по умолчанию; движется только сверхъестественное. Технически это значит: моушн сосредоточен в слое эффектов и микровзаимодействий, а контент остаётся стабильным. Никакого тотального parallax.

- **Декларативные ревилы и жесты —** `motion/react` (reveal, layout, hover).
- **Оркестрация скролл-сцен —** GSAP + ScrollTrigger; кинетика зачинов рассказов — SplitText (переписан, доступен, лёгкий).
- **Текстура «офиса» —** WebGL-оверлей зерна/CRT-фликера (R3F) ИЛИ CSS/SVG-шум как лёгкий фолбэк; единственный обязательный WebGL-элемент.
- **Haunted-элементы —** кастомный курсор, «неправильный» скроллбар, глитч-типографика, проявление из тумана — Motion/CSS/canvas.

**Связка Lenis ↔ ScrollTrigger (только в «Витрине»).**

```js
// (vitrina)/layout — инициализация один раз
const lenis = new Lenis();
lenis.on('scroll', ScrollTrigger.update);
gsap.ticker.add((time) => lenis.raf(time * 1000));
gsap.ticker.lagSmoothing(0);
// «Читальня» Lenis НЕ инициализирует — нативный скролл для длинного чтения.
```

**Единый шлюз reduced-motion.**

```ts
// components/motion/useReducedMotionSafe.ts
// Источник правды: ОС-настройка prefers-reduced-motion + ручной тумблер «снизить эффекты».
// ВСЕ анимации, грейн, фликер, parallax обязаны спрашивать этот хук
// и иметь полноценный статический сценарий (а не просто «выключено»).
```

## 8. Каталог компонентов и контракты

Ключевые компоненты и их зоны ответственности. Контракты — ориентир для типизации.

- **`<Wheel />`** — пропсы: `stories, themes, activeStoryId?, onSelectStory, onSelectTheme`. Внутри — симуляция d3-force; рендер SVG; клавиатура (roving tabindex); делегирует доступность на `<WheelIndex />`.
- **`<WheelIndex />`** — семантический список «тема → рассказы»; каноническая и полностью доступная навигация; на мобильном — основной вид Колеса.
- **`<ThemeNode />` / `<StoryNode />`** — узлы графа; состояния hover/focus/active; на узле-рассказе — превью firstLine.
- **`<Reader />`** — пропсы: `book, story, content (MDX), onProgress`. Пагинация, плашки иллюстраций, прогресс.
- **`<BookSwitcher />`** — переключение между тремя книгами в Читальне.
- **`<IllustrationPlate />`** — editorial-размещение графики Кучеренко (с воздухом, сохранением палитры).
- **`<MotionProvider />` + `useReducedMotionSafe()`** — единый шлюз моушена; гейтит все эффекты; хранит ручной тумблер.
- **`<GrainOverlay />`** — WebGL/canvas-зерно и фликер; ленивая загрузка; отключается под reduced-motion / save-data / low-power.
- **`<CustomCursor />` / `<GlitchText />` / `<FogReveal />`** — haunted-микровзаимодействия; все — с reduced-motion-фолбэком.

## 9. Типографика: дуальная система и шрифты с кириллицей

**Концепция в коде.** Две гарнитуры выражают «душу против системы». Жёсткое ограничение: обе обязаны иметь полноценную кириллицу (многие выразительные дисплейные шрифты её не имеют — см. 11-D1).

- **Проза («душа») —** гуманистическая антиква с кириллицей и вариативностью: Source Serif 4, Lora или Spectral (все с кириллицей, OFL).
- **Система/UI («машина») —** холодный гротеск Inter (отличная кириллица, вариативный) и моноширинный акцент JetBrains Mono или IBM Plex Mono (кириллица есть).
- **Подключение —** `next/font`: сабсет `cyrillic+latin`, preload для прозы, `font-display: swap`; следить за CLS вариативных файлов.
- **Токены —** размеры/межстрочье/цвета — через `@theme` Tailwind v4 в OKLCH; единый ритм для Витрины и Читальни.
- **Проверка —** рендер «», тире и ударений (во́ду) в выбранных шрифтах до фиксации выбора.

## 10. Доступность и производительность

**Доступность (часть определения готовности).**

- **reduced-motion-паритет —** полноценная статическая версия всего опыта, а не отключённая анимация.
- **Клавиатура и семантика —** Колесо навигируется с клавиатуры; видимый фокус; ARIA; индекс-двойник как канон.
- **Контраст —** тексты проходят WCAG AA даже в «офисной» палитре; OKLCH упрощает контроль контраста токенов.
- **Без вспышек —** фликер/глитч — мягкие и гейтятся; никаких >3 вспышек/сек (WCAG 2.3.1).

**Производительность.**

- **SSG/ISR контента;** острова интерактива через next/dynamic (`ssr:false` для WebGL и Колеса).
- **React Compiler (Next 16) —** автомемоизация, меньше ререндеров без ручных useMemo.
- **Изображения —** next/image, AVIF/WebP; плашки Кучеренко — с размерами, без layout shift.
- **Шрифты —** сабсет, preload, swap; вариативные файлы взвешивать в бюджете.
- **SEO/шеринг —** SSR/SSG контент-маршрутов; per-story метаданные; динамические OG (next/og) в стиле офисной готики; sitemap (на Next 16 — async generateSitemaps).
- **Бюджеты —** целевой Lighthouse и Core Web Vitals (LCP/CLS/INP) фиксируются заранее (раздел 13).

## 11. Узкие места и вероятные трудности

Самая важная часть документа: где проект ломается на практике и как это предотвратить. Сгруппировано по областям.

### A. Контент и ассеты

**A1. Пайплайн из PDF — главный риск всего проекта.**
- **Суть.** Книги — печатные экспорты InDesign (PDF/X). Чистое структурирование (границы рассказов, курсив, сцены `***`, «», ударения, эпиграфы на синдарине/латыни) трудозатратно: переносы, мягкие дефисы, лигатуры, колонтитулы и номера страниц вперемешку с текстом.
- **Митигация.** Полуавтоматический `ingest-pdf.ts` + обязательная ручная вычитка каждого рассказа; typograf на сборке; иллюстрации извлекать отдельно (pdfimages) в исходном разрешении. Заложить на это реальное время до старта фронтенда.

**A2. Права на ассеты.**
- **Суть.** Иллюстрации Кучеренко и тексты — авторские; нужно подтверждённое право веб-использования. GSAP бесплатен, но free-to-use, не OSS.
- **Митигация.** Зафиксировать с автором/иллюстратором права на веб; шрифты брать под OFL; ИИ-ки-вижуал из v1 не используется (от него отказались).

**A3. Обновление «Мастерской».**
- **Суть.** 3-я книга растёт во времени; автор может не хотеть править MDX/Git.
- **Митигация.** Канон — файловый (стабилен). Для «Мастерской»/контактов — опциональная CMS (Payload/Sanity), подключается точечно; это добавляет инфраструктуру, поэтому только при реальной потребности.

### B. Колесо (граф)

**B1. Доступность графа.**
- **Суть.** Пространственный граф плохо доступен с клавиатуры и для скринридеров — риск красивого, но недоступного хаба.
- **Митигация.** Канон — доступный `<WheelIndex />` (список «тема → рассказы»); визуальный граф — улучшение поверх. Roving tabindex, стрелки, видимый фокус, ARIA.

**B2. Перестройка Колеса: предсказуемость и перфоманс.**
- **Суть.** Повторный прогон симуляции при чтении заставляет узлы «прыгать» (дезориентация) и может тормозить на слабых устройствах.
- **Митигация.** Детерминированный seed, ограниченные итерации, симуляция в Web Worker или предрасчёт; плавные переходы позиций; под reduced-motion — без reflow, только подсветка.

**B3. Колесо на мобильном.**
- **Суть.** 44 узла не помещаются на экран 380px.
- **Митигация.** На мобильном Колесо вырождается в индекс/список (или упрощённый радиал с pan/zoom). Отдельный мобильный UX, а не масштабирование.

### C. Движение и скролл

**C1. Конфликт скролл-движков.**
- **Суть.** Lenis + ScrollTrigger + скролл Motion + нативный скролл дерутся: джанк, неверные позиции, сломанные якоря; в Читальне инерционный скролл делает чтение 150 страниц мучительным.
- **Митигация.** Lenis только в `(vitrina)`; Читальня — нативный скролл; связка Lenis↔ScrollTrigger по документации (раздел 7). Lenis ИЛИ ScrollSmoother, но не оба.

**C2. Бюджет WebGL/R3F.**
- **Суть.** Даже «минимальный» WebGL (зерно/CRT, 3D-книги) — вес бандла, расход батареи, проблемы мобильных GPU; риск противоречия обещанию «перфоманс/доступность прежде всего».
- **Митигация.** WebGL — один опциональный оверлей зерна/фликера; ленивая загрузка; отключение под reduced-motion/save-data/low-power; CSS/SVG-фолбэк. Книги — 2.5D-трансформации вместо 3D, если хватает.

**C3. Фоточувствительность и вестибулярные расстройства.**
- **Суть.** Фликер/глитч и parallax могут провоцировать приступы и тошноту.
- **Митигация.** Строгий reduced-motion; никаких быстрых вспышек (WCAG 2.3.1); фликер мягкий и гейтится; глобальный тумблер «снизить эффекты» сверх ОС-настройки.

### D. Типографика и доступность контента

**D1. Шрифты с кириллицей — реальное ограничение.**
- **Суть.** Многие выразительные дисплейные/editorial-гарнитуры не имеют кириллицы — дуальная система рушится, если влюбиться в латиницу-only шрифт.
- **Митигация.** Выбирать из кириллических семейств сразу (раздел 9); сабсет через next/font; следить за FOUT/CLS; проверить «» и ударения. Вариативные файлы с полной кириллицей крупнее — учесть в бюджете.

**D2. «Офисная готика» против читаемости и контраста.**
- **Суть.** Институциональные серые и приглушённый свет легко проваливают WCAG-контраст для текста и «мутят» картинку.
- **Митигация.** Высокий контраст и читаемость — на реальных поверхностях чтения (Читальня); мрачная палитра — для атмосферы, не для длинного текста; токены контраста проверять (OKLCH помогает). Эффекты не должны снижать читаемость контента.

### E. Производительность и архитектура

**E1. SSG против интерактивности (гидратация).**
- **Суть.** Колесо и моушн — тяжёлые клиентские компоненты, а контент статичен. Риск больших бандлов, дорогой гидратации и JS-перегруженной главной (Колесо).
- **Митигация.** SSG контента; островная архитектура; next/dynamic `ssr:false` для WebGL и Колеса; React Compiler; PPR/стриминг; первый кадр Колеса — SSR-SVG с последующим JS-улучшением.

**E2. Состояние чтения и дип-линки между средами.**
- **Суть.** Прогресс, последняя позиция, дип-линки из Колеса, навигация назад/вперёд между «Витриной» и «Читальней» (разные layout-группы), View Transitions между группами.
- **Митигация.** localStorage/IndexedDB для прогресса; стабильные slug; View Transitions API (Next 16) для перехода узел→страница; контроль scroll restoration.

**E3. SEO и шеринг иммерсивного JS-сайта.**
- **Суть.** Тяжёлые JS-сайты проседают в SEO и соц-превью.
- **Митигация.** SSR/SSG контент-маршрутов; per-story метаданные; динамические OG (next/og) — рендер первой строки на «офисной» карточке; sitemap (async generateSitemaps на Next 16); структурированные данные книг/автора.

### F. Процесс

**F1. Объём ручной типографики.**
- **Суть.** «Первая строка как событие», кинетика зачинов (SplitText), editorial-размещение иллюстраций — это ручной крафт × 34 рассказа; риск разрастания объёма.
- **Митигация.** Небольшой набор переиспользуемых «шаблонов зачина», параметризованных под рассказ; не верстать 34 уникальные сцены вручную.

## 12. Рекомендованная последовательность реализации

Порядок снижает риск: сначала контент и менее рискованная Читальня, доступный индекс — до визуального графа, атмосфера — в конце.

- **Фаза 0 — Контент-пайплайн.** ingest-pdf.ts, схема + zod, вычитка рассказов, извлечение иллюстраций. Блокирует всё остальное (см. 11-A1).
- **Фаза 1 — Каркас и токены.** Next 16 + Tailwind v4 @theme (OKLCH), дуальная типографика, два layout'а (Витрина/Читальня), MotionProvider + reduced-motion-шлюз.
- **Фаза 2 — Читальня.** Наивысшая контентная ценность при меньшем риске: Reader, пагинация, плашки, прогресс, переключение книг.
- **Фаза 3 — Колесо.** Сначала доступный `<WheelIndex />` (канон), затем визуальный граф d3-force поверх; перестройка при чтении.
- **Фаза 4 — Атмосфера и haunted-UI.** Lenis-сцены, GSAP/SplitText зачины, грейн/CRT-оверлей, кастомный курсор, глитч — всё под reduced-motion.
- **Фаза 5 — Мастерская и контакты.** Раздел 3-й книги (опц. CMS), тихий финал.
- **Фаза 6 — Закалка a11y/перфоманс.** Аудит контраста и клавиатуры, бюджеты Lighthouse/CWV, OG/sitemap, мобильные сценарии.

## 13. Критерии готовности (Definition of Done)

Сайт считается готовым, когда выполнены все пункты — это страховочные рельсы концепции.

- **Reduced-motion-паритет:** при включённой настройке доступен полноценный статический опыт (включая Колесо и чтение).
- **Доступность:** WCAG AA по контрасту текста; Колесо навигируется с клавиатуры; индекс-двойник работает; видимый фокус и ARIA.
- **Кириллица:** проза, UI, «», тире и ударения рендерятся корректно в выбранных вариативных шрифтах; нет FOUT-скачков.
- **Скролл:** нет конфликтов движков; Читальня — нативный скролл; якоря и восстановление позиции работают.
- **Производительность:** целевые Lighthouse и Core Web Vitals достигнуты; WebGL ленив и отключаем; контент — SSG/ISR.
- **Контент:** все 34 рассказа вычитаны и структурированы; иллюстрации привязаны; метаданные и OG на месте.
- **Архитектура:** две среды разделены; дип-линки Колесо↔Читальня и переходы между группами стабильны.

## 14. Пошаговый план реализации для Claude Code

Этот раздел операционализирует §12: разбивает фазы на конкретные шаги с готовыми промптами. Каждый промпт (блок кода) вставляется в Claude Code по одному; перед переходом к следующему шагу проверяется приёмка.

### 14.0. Как вести проект в Claude Code

- **Положить оба документа в репозиторий —** папка `/docs`: концепция (что строим) и эта спецификация (как строим). Claude Code читает их как контекст; промпты ниже ссылаются на разделы по номерам (§4, §11-A1 и т.д.).
- **Завести CLAUDE.md в корне —** постоянная память проекта и страховочные рельсы. Claude Code подхватывает её автоматически. Шаблон ниже.
- **Работать пофазно —** коммит после каждого шага; не переходить к следующему до выполнения приёмки.
- **Тяжёлые/рискованные шаги —** сначала план, потом код. В промптах для PDF-пайплайна и графа явно сказано «покажи план, дождись подтверждения».
- **Версии —** пинить актуальный stable; перед установкой сверять `npm view <pkg> version`.
- **Доступность и reduced-motion —** гейты, а не «потом»: они зашиты в критерии приёмки шагов и в §13.

**Шаблон CLAUDE.md (положить в корень репозитория).**
