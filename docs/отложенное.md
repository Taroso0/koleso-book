## Закалка: функциональные баги Reader (вне Шага 6.1 — отдельный шаг)

Выявлены в `docs/проверка-в-браузере.md`. Это НЕ доступность — чиним отдельным шагом
закалки, чтобы не размывать a11y-аудит 6.1:

- [x] **Постраничный размер «прыгает»** — закрыто в `3f218f3` (B2): высота страницы
      берётся из CSS (`h-[calc(100svh-9rem)]`, `ReaderShell.tsx:204`), `measure()`
      читает только ширину колонки, при входе в режим — `scrollTo(0, 0)`.
- [x] **Мелькание плашки в paged** — закрыто там же (B4): `object-fit: contain` +
      `break-after: column`.
- [x] **Сосед открывается с последней страницы** — закрыто там же (B3): `key={story.slug}`
      на `ReaderShell` (`components/reader/Reader.tsx:41`) сбрасывает состояние оболочки.
- [ ] **Reduced-effects: 1 кадр движения при F5.** Ручной тумблер читается из
      localStorage в эффекте → возможно мгновение анимации до применения. reduced-motion-
      ПАРИТЕТ при этом цел (итог верный). Фикс — синхронная инициализация без
      hydration-mismatch; отложено как низкий приоритет.
- [ ] **Рассказ короче экрана не отмечается прочитанным.** В «свитке» прогресс считается
      как `scrollY / (scrollHeight − innerHeight)`; если текст помещается на экран,
      знаменатель ≤ 0 → `pct` всегда 0, и «Колесо» такой рассказ не запомнит
      (`ReaderShell.tsx`, эффект скролла). Ни один из 34 рассказов сейчас так не выглядит,
      но дыра реальна. Тот же случай в постраничном режиме при `pageCount === 1`.

- [ ] **Условия приёмки третьей книги.** До добавления `kniga3` в «Читальню» обязательно:
