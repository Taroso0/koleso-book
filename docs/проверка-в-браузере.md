# Ручная проверка в браузере (Фазы 0–2)

Что нужно проверить **глазами и руками** — то, что нельзя подтвердить сборкой.

## Запуск

```bash
npm run dev        # http://localhost:3000
```
Проверять в двух режимах ширины: **десктоп** и **мобильный** (DevTools → device toolbar,
напр. 360–390px). Современный браузер (Chrome/Edge/Firefox/Safari).

---

## 1. Шрифты и кириллица (Фаза 1) — страница `/`

- [ ] Текст-специмен рендерится тремя гарнитурами: антиква (проза), гротеск (система), моно.
- [ ] Корректны: ёлочки «…», короткое и длинное тире (– —), многоточие …, **ударение во́ду**.
- [ ] Нет «тофу» (□) и пропавших глифов ни в одной из трёх гарнитур.

## 2. Две среды и движение

- [ ] `/` (Витрина) — есть плавный (инерционный) скролл, если страница прокручивается.
- [ ] `/read` (Читальня) — скролл **нативный**, без инерции/«доезжания».
- [ ] Тумблер «снизить эффекты» на `/` нажимается (меняется вид/`aria-pressed`).
- [ ] Включите системный *prefers-reduced-motion* — поведение остаётся корректным.

## 3. Индекс «Читальни» — `/read`

- [ ] Две книги: «Колизей» (2022) и «Колесо Сансары» (2023).
- [ ] В каждой ровно **17** рассказов, порядок и нумерация верные.
- [ ] Ссылки на книгу и на рассказ открываются.
- [ ] Карточка «Продолжить чтение» — появляется **после** того, как откроете рассказ
      и немного проскроллите (см. §7).

## 4. Все 34 рассказа открываются и читаются

Пройдитесь по обеим книгам (можно через prev/next внизу рассказа).

- [ ] Каждый рассказ открывается по своему слагу; заголовок верный.
- [ ] Микротипографика в тексте: ёлочки, тире, … — без «прямых» кавычек и без `...`.
- [ ] Разделители сцен показаны как `* * *`.
- [ ] Курсив (*наклонный*) отображается там, где он есть (напр. сноски-расшифровки в конце).
- [ ] Навигация «Назад / Дальше» внизу ведёт к соседним рассказам той же книги.
- [ ] Несуществующий адрес (напр. `/read/koleso/нет-такого`) даёт **404**.

## 5. Иллюстрации-плашки (§6)

- [ ] У каждого рассказа есть плашка-иллюстрация, по центру, «с воздухом».
- [ ] **Нет «прыжка» вёрстки** при загрузке картинки (место зарезервировано).
- [ ] ⚠️ **Главное, что может только человек:** это действительно иллюстрация рассказа,
      а не отсканированная страница с текстом и не чужая/служебная картинка.
      Если где-то картинка «не та» — отметьте слаг, поправим извлечение.

## 6. Переключатель книг (BookSwitcher)

- [ ] На странице рассказа и на странице книги есть переключатель из 2 книг.
- [ ] Текущая книга подсвечена; клик уводит в другую книгу.
- [ ] (Третьей книги «Мастерская» пока **нет** — так задумано, см. «не баг».)

## 7. Прогресс чтения (Шаг 2.3)

- [ ] Откройте рассказ, проскролльте до середины, уйдите на `/read`, вернитесь в рассказ —
      позиция **восстановилась**.
- [ ] На `/read` карточка «Продолжить» указывает на последний рассказ и верный **%**.
- [ ] Полоска прогресса в карточке соответствует прочитанному.
- [ ] Перезагрузка страницы (F5) — позиция тоже восстанавливается.

## 8. Постраничный режим (Шаг 2.3)

Внизу рассказа — панель «Режим: Свиток / Страницы».

- [ ] Переключение на «Страницы» разбивает рассказ на страницы.
- [ ] Кнопки ← / → и **клавиши стрелок** листают; индикатор «N / всего» меняется.
- [ ] Работает на **мобильной** ширине (узкий экран).
- [ ] Заголовок и плашка не рвутся пополам между страницами.
- [ ] Длинные рассказы листаются до конца (последняя страница достижима).
- [ ] Переключение «Страницы ↔ Свиток» сохраняет примерную позицию.
- [ ] При *reduced-motion* листание — **без анимации** (мгновенно).

## 9. Темы чтения (Шаг 2.3)

Панель «Тема: Светлая / Сепия / Тёмная».

- [ ] Все три темы переключаются; текст читаем в каждой (комфортный контраст).
- [ ] Выбор **сохраняется** при переходе между рассказами.
- [ ] После F5 тема применяется **без вспышки** другой темы (проверьте на Сепии/Тёмной).
- [ ] Тема чтения влияет только на «Читальню»; на `/` (Витрина) — не влияет.

## 10. Доступность (клавиатура)

- [ ] Tab проходит по ссылкам/кнопкам; **фокус видимый** (натриевое свечение).
- [ ] В постраничном режиме стрелки ← / → листают.
- [ ] Масштаб браузера 200% — вёрстка не ломается, текст не обрезан.

## 11. Адаптивность

- [ ] Десктоп: широкие поля, узкая колонка текста — читать комфортно.
- [ ] Мобильный (~360px): поля и размеры адекватные, ничего не вылезает за экран.
