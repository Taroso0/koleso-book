Снимает скриншоты ключевых состояний сайта и даёт их осмотреть инструментом Read — вместо цикла «пользователь смотрит → скриншот в Downloads → правка».

## 1. Поднять dev-сервер (если ещё не работает)

Проверь порт 3000. Если занят призрачным процессом с прошлой сессии — убей его:
```powershell
Get-NetTCPConnection -LocalPort 3000 -State Listen -ErrorAction SilentlyContinue |
  ForEach-Object { Stop-Process -Id $_.OwningProcess -Force }
```
Запусти `npm run dev` в фоне (run_in_background), дождись в выводе `Ready in`.
Если Turbopack паникует «Failed to restore task data (corrupted database)» — удали `.next` и перезапусти.

## 2. Снять скриншоты

```bash
node .claude/skills/verify-in-browser/screenshot.mjs
```
- `--only home,story` — только часть состояний (подстрока имени, через запятую);
- `--base http://localhost:3001` — если dev-сервер на другом порту.

Состояния: `home` (+reduced-motion, +mobile), `workshop`, `contacts`, `read-index`, рассказ в трёх темах чтения (light/sepia/dark), `story-dark-flash` (кадр сразу после навигации — ловит вспышку светлой темы на F5), `story-mobile`. Playwright и chromium уже установлены (зависимость `.ds-sync`); если chromium пропал — `cd .ds-sync && npx playwright install chromium`.

## 2а. Части сайта и состояния «ниже сгиба» (грабли Витрины — важно)

`screenshot.mjs` снимает по вьюпорту — годится для того, что видно сразу. Но на Витрине инерционный скролл **Lenis** откатывает программный `scrollIntoView`, поэтому обычный вьюпорт-снимок состояния **ниже сгиба** (граф «Колеса», список тем, скролл-сцены хаба) падает обратно на hero. Не борись с Lenis (эмуляция `reduced` тоже ненадёжна) — снимай **сам элемент** через `element-shot.mjs`.

**Не угадывай селектор — бери часть из каталога.** У `element-shot.mjs` есть карта всех частей сайта: `--part <имя>` сам откроет нужную страницу и снимет верный элемент с первого раза.

```bash
node .claude/skills/verify-in-browser/element-shot.mjs --list          # печатает всю карту
node .claude/skills/verify-in-browser/element-shot.mjs --part wheel-index
```

Карта частей (актуальный список — всегда через `--list`; при рефакторинге разметки сверяй селекторы с кодом):

| `--part` | Страница | Что это |
|---|---|---|
| `hero` | `/` | Первый экран «Тёплое окно» (h1, горящее окно, тизер Колеса) |
| `wheel-graph` | `/` | Визуальный граф «Колеса» (d3-force + SVG) — **только десктоп** |
| `wheel-index` | `/` | Доступный список «тема → рассказы» — **канон навигации** (свёрнутые `<details>`) |
| `author` · `books` · `workshop-teaser` | `/` | Тизер-секции хаба |
| `home-footer` | `/` | Подвал: «Войти в Читальню» + тумблер «снизить эффекты» + Контакты |
| `workshop` | `/workshop` | Лента записей Мастерской |
| `contacts` | `/contacts` | Тихий финал (ночная палитра) |
| `read-index` | `/read` | Оглавление Читальни |
| `reader` · `story-article` · `reader-toolbar` | `/read/<книга>/<рассказ>` | Рассказ целиком · тело · панель настроек чтения (`:story` = первый найденный рассказ) |

Прочие опции: `--url`, `--reduced`, `--mobile`, `--name`, `--base`, `--settle`, `--selector` (ad-hoc, если части нет в каталоге). Десктоп-острова (`wheel-graph`) на `--mobile` спрятаны `display:none` — скрипт это сообщит и не упадёт (норма).

**Интерактив — кликом или фокусом.**
- **Раскрыть/переключить** (свёрнутый `<details>`, меню) — `--click '<css>'` (можно несколько раз, кликает по очереди) кликает перед снимком:
  ```bash
  node .claude/skills/verify-in-browser/element-shot.mjs --part wheel-index \
    --click 'summary:has(#theme-death)' --name wheel-death-open
  ```
- **Hover/«живое внимание»** — задавай **ФОКУСОМ**, не мышью: при element-screenshot Playwright доскроллит элемент, мышь съезжает → `mouseleave` → hover теряется. Фокус липкий и переживает доскролл:
  ```bash
  node .claude/skills/verify-in-browser/element-shot.mjs --part wheel-graph \
    --focus '[aria-label="Тема «Смерть»"]' --name wheel-focus-death
  ```

## 3. Осмотреть и сверить

Прочитай каждый PNG инструментом Read и проверяй против жёстких правил CLAUDE.md:
- тёмная/сепия темы читальни применились (нет белой вспышки в `story-dark-flash`);
- reduced-motion — паритет: полноценный статический вид, а не «пусто/выключено»;
- контраст, кириллица, отсутствие поехавшей вёрстки на 390px;
- граф «Колеса» отрисован (узлы и связи видны).

Интерактив и состояния ниже сгиба снимай через `element-shot.mjs` (см. §2а): `--part` находит элемент, `--click`/`--focus` задают состояние. Если нужен сценарий, которого они не покрывают (длинная последовательность действий, зачин при переходе из «Колеса») — пиши одноразовый скрипт рядом по образцу `screenshot.mjs`/`element-shot.mjs` (playwright через `createRequire` из `.ds-sync`), не изобретай обвязку заново.

## 4. Прибраться и отчитаться

Если dev-сервер поднимал ты — останови его. В отчёте: что проверено, что подтвердилось, что расходится (с приложением путей к снимкам). Финальную live-проверку глазами всё равно оставь пользователю — ручной чек-лист `docs/проверка-в-браузере.md`.
