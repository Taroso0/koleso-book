Снимает скриншоты ключевых состояний сайта и даёт их осмотреть инструментом Read — вместо цикла «пользователь смотрит → скриншот в Downloads → правка».

## 1. Поднять dev-сервер (если ещё не работает)

Проверь порт 3000. Если занят призрачным процессом с прошлой сессии — убей его:
```powershell
Get-NetTCPConnection -LocalPort 3000 -State Listen -ErrorAction SilentlyContinue |
  ForEach-Object { Stop-Process -Id $_.OwningProcess -Force }
```
Запусти `npm run dev` в фоне (run_in_background), дождись в выводе `Ready in`.
Если Turbopack паникует «Failed to restore task data (corrupted database)» — удали `.next` и перезапусти.

## 2. Снять скриншоты

```bash
node .claude/skills/verify-in-browser/screenshot.mjs
```
- `--only home,story` — только часть состояний (подстрока имени, через запятую);
- `--base http://localhost:3001` — если dev-сервер на другом порту.

Состояния: `home` (+reduced-motion, +mobile), `workshop`, `contacts`, `read-index`, рассказ в трёх темах чтения (light/sepia/dark), `story-dark-flash` (кадр сразу после навигации — ловит вспышку светлой темы на F5), `story-mobile`. Playwright и chromium уже установлены (зависимость `.ds-sync`); если chromium пропал — `cd .ds-sync && npx playwright install chromium`.

## 3. Осмотреть и сверить

Прочитай каждый PNG инструментом Read и проверяй против жёстких правил CLAUDE.md:
- тёмная/сепия темы читальни применились (нет белой вспышки в `story-dark-flash`);
- reduced-motion — паритет: полноценный статический вид, а не «пусто/выключено»;
- контраст, кириллица, отсутствие поехавшей вёрстки на 390px;
- граф «Колеса» отрисован (узлы и связи видны).

Для интерактивных состояний (hover узла графа, зачин при переходе из «Колеса») — напиши одноразовый скрипт рядом по тому же образцу (playwright через `createRequire` из `.ds-sync`), не изобретай обвязку заново.

## 4. Прибраться и отчитаться

Если dev-сервер поднимал ты — останови его. В отчёте: что проверено, что подтвердилось, что расходится (с приложением путей к снимкам). Финальную live-проверку глазами всё равно оставь пользователю — ручной чек-лист `docs/проверка-в-браузере.md`.
