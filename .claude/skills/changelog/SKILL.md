# Запись в журнал изменений (ИЗМЕНЕНИЯ.md)

Журнал `ИЗМЕНЕНИЯ.md` — человекочитаемый дневник для владельца проекта: он
следит за развитием по нему, не читая код. Заполняется после каждой задачи с правками кода/настроек.

## Что изменилось в этой задаче (живой контекст)

- Файлы: !`git diff --stat HEAD`
- Незакоммиченное: !`git status --short`
- Стиль прошлых записей: !`git log --oneline -5`

## Задача

Составь и добавь запись о правках, относящихся к запросу `$ARGUMENTS` (если пусто — опиши текущие
изменения из контекста выше).

### 1. Формат записи (как в шапке `ИЗМЕНЕНИЯ.md`)

```markdown
## ГГГГ-ММ-ДД — Короткий заголовок простыми словами

**Что изменилось:** простыми словами, без терминов — что стало работать иначе.

**Зачем:** какую задачу или проблему это решает.

**Технические подробности:** какие файлы и как затронуты (для разработчика; можно пропустить).

**Как проверить:** что открыть/нажать, чтобы убедиться, что всё работает.
```

- Дату бери сегодняшнюю.
- Если какой-то пункт не применим — опусти его (так принято в файле).
- Тон — спокойный, понятный без знания кода. Без жаргона в «Что изменилось»/«Зачем»/«Как проверить»;
  термины и имена файлов — только в «Технические подробности».

### 2. Куда вставить

Новые записи — **сверху**: прочитай начало `ИЗМЕНЕНИЯ.md`, найди разделитель `---`
перед самой свежей датированной записью (`## ГГГГ-ММ-ДД …`) и вставь новый блок **над ней**
(с разделителем `---` после своего блока, как у соседних записей). Не трогай вступление и раздел
«Как читать запись».

### 3. После вставки

Коротко подтверди, какой заголовок добавлен. Запись входит в тот же коммит, что и правки задачи
(конвенция проекта: один логический кусок — один коммит).

## Подводные камни

- Не дублируй технический changelog из git — журнал именно про «человеческий» смысл правки.
- Одна задача — одна запись (несколько мелких правок можно объединить в один блок с подпунктами).
- Не выдумывай «Как проверить»: опиши реально доступную проверку (страница сайта на
  `http://localhost:3000`, конкретный раздел — «Читальня», «Колесо», «Мастерская», — тема чтения,
  или команда `/verify` / `/verify-in-browser`).
