Аргумент — путь к папке хендоффа. Без аргумента: найди самую свежую папку `design_handoff_*` внутри `C:\Users\Sasha\Downloads\Боковым зрением — дизайн-система*\` (по LastWriteTime) и **подтверди выбор у пользователя**, прежде чем работать.

## Процедура

1. **Прочитай все файлы хендоффа полностью** (спека, токены, ассеты) — не по диагонали.
2. **Сначала сверь спеку с текущим кодом.** Часть может быть уже реализована — прецедент `night_polish` v2, где правок не потребовалось вовсе. Выпиши явно: что уже есть / что расходится / чего нет.
3. **План — только по реальным расхождениям.** При конфликте спеки с жёсткими правилами CLAUDE.md или концепцией (`docs/концепция.md`) — остановись и спроси, не отклоняйся молча.
4. **Реализуй.** Держись дизайн-токенов `app/globals.css` (OKLCH), reduced-motion-паритета и разделения двух сред (vitrina / chitalnya).
5. **Проверка:** `/verify`; для визуальных изменений — `/verify-in-browser`.
6. **Чек-лист:** если появились новые проверяемые состояния — добавь их в `docs/проверка-в-браузере.md` (по образцу существующих секций).
7. **Коммит** по конвенции проекта: русское сообщение, один логический кусок — один коммит. `git push` — только по явной просьбе.

## Финальный отчёт
Три списка: реализовано · уже было в коде · осталось за пользователем (live-проверка глазами). Плюс раздел чек-листа, который нужно пройти вручную.
