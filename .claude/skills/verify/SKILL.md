## 1. Типы

```bash
npx tsc --noEmit
```
Должно быть 0 ошибок.

## 2. Линт
```bash
npm run lint
```
Линтит весь репозиторий: сгенерированный `ds-bundle/`, тулчейн `.ds-sync/` и
`coverage/` вынесены в `globalIgnores` (`eslint.config.mjs`), поэтому шума нет.

Baseline чист — **0 проблем** (ни ошибок, ни предупреждений). Прежние 8
(react-hooks/refs, set-state-in-effect, preserve-manual-memoization в `WheelCanvas`)
исправлены; техдолга больше нет.

**Провал шага = любая ошибка или предупреждение.** Если вернулась ошибка
`react-hooks/*` — это регресс, чинить, а не добавлять в «известные».

## 3. Тесты
```bash
npm test
```
Vitest + jsdom. Baseline — **16 файлов, 177 тестов, все зелёные** (~9 с).
Провал шага = любой упавший тест или падение числа тестов без причины.

Покрытие проверять отдельно (`npm run test:coverage`, порог 90%) — при правках
в области покрытия: `lib/`, схемы и загрузчики контента, интерактивные
компоненты (`WheelIndex`, `ContinueReading`, `MotionProvider`,
`useHauntedCapability`). Визуал, WebGL и анимации в покрытие не входят —
для них шаг пропускается, проверка идёт через `/verify-in-browser`.

## 4. Прод-сборка
```bash
npm run build
```
Заодно валидирует контент (zod): невалидный frontmatter ломает сборку с указанием файла и поля. Вывод длинный — смотри хвост (`Select-Object -Last 30` / `tail -30`).
