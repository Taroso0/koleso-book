# Сайт Евгения Кирилова — «Боковым зрением»

## Что это
Персональный авторский сайт писателя. Статический контент-сайт (Next.js App Router,
полная SSG) с «островами» тяжёлой клиентской интерактивности. Бэкенда и БД нет.
Концепция (v2.0): «офисная готика» (жуткое и чудесное внутри обыденного, «одержимый»
интерфейс) · «Колесо» — живая карта смыслов как дом и навигация · типографика-герой
(«душа против системы»). Метафора сайта — чудо видно «боковым зрением».

**Статус:** сайт построен, план стройки выполнен; режим доработок без фиксированного плана.

## Карта репозитория
Алиас импорта: `@/*` → корень.

- `app/(vitrina)` и `app/(chitalnya)` — **две среды**, см. рельсы ниже
- `lib/` — вся чистая логика: контент, граф «Колеса», укладка, типографика, SEO
- `content/` — данные и `schema.ts` (все zod-схемы, канонические `THEME_IDS`/`BOOK_IDS`)
- `components/` — `wheel/` · `reader/` · `motion/` (шлюз движения) · `haunted/` (атмосфера)
- `scripts/` — Python-пайплайн контента и node-гейты сборки
- `tests/` — Vitest, порог покрытия 90 % по заявленной области
- Не рантайм-код: `ds-bundle/`, `.ds-sync/`, `.design-sync/` (экспорт дизайн-системы),
  `content/_raw/` (промежуточный этап пайплайна)

Подробности и причины — `docs/решения.md`.

## Стек (пинить актуальный stable; сверять `npm view <pkg> version`)
Next.js 16 (App Router) · React 19 · TypeScript 5 · Tailwind v4 (CSS-first, OKLCH) ·
Motion (motion/react) · GSAP 3.13+ · Lenis · d3-force · R3F v9 (точечно) ·
gray-matter + zod · next/font (cyrillic+latin). Node 20+.
Не выдумывать API: не уверен — проверить (`npm view`) или спросить.

⚠️ **MDX-компилятора в проекте НЕТ**, хотя файлы рассказов лежат с расширением `.mdx`
(исторический артефакт). Ни `@next/mdx`, ни `@mdx-js/*` не установлены: gray-matter
отделяет frontmatter, а тело рендерит `components/reader/ProseBody.tsx` — **абзацы по
пустой строке, `*курсив*` и `***` как разделитель сцен, и всё**. Заголовки, списки,
ссылки, JSX отрендерятся буквальным текстом.

## Команды
```bash
npm run dev          # dev-сервер, http://localhost:3000
npm run build        # прод-сборка (SSG); заодно zod-валидация контента:
                     # невалидный frontmatter ломает сборку с указанием файла и поля
npm run check:size   # гейт на вес страниц (после build)
npm run typecheck    # tsc --noEmit
npm test             # vitest run
npm run test:coverage  # покрытие, порог 90% (падает, если ниже)
npm run lint         # линт всего репозитория; baseline — 0 проблем
```
`npm run lint` даёт **0 проблем**: генерированный `ds-bundle/`, `.ds-sync/` и
`coverage/` вынесены в `globalIgnores`. Провал = любая ошибка ИЛИ предупреждение;
вернувшаяся `react-hooks/*` — регресс, а не «известная».

**CI** (`.github/workflows/ci.yml`) — на каждый push в main и каждый PR. Все шаги
блокирующие: типы → тесты с покрытием → сборка → **вес страниц** → линт.
Чего в CI НЕТ: визуальных проверок (Playwright живёт в незакоммиченном `.ds-sync/`)
и любой проверки Python-пайплайна.

Контент-пайплайн (Python 3, `pip install pymupdf pillow`; на сборке сайта не нужен):
```bash
python scripts/ingest_pdf.py    # source/*.pdf → content/_raw/<book>/*.md
                                #   существующие файлы НЕ трогает (там ручная вычитка);
                                #   --force перезаписать, --dry-run показать без записи
python scripts/build_mdx.py     # _raw → content/stories/*.mdx + плашки WebP
                                #   сохраняет ручные themes[] и галочки чек-листа
python scripts/illustrations_manifest.py   # размеры плашек → content/illustrations.json
```
Единственный необратимый шаг — `ingest_pdf.py --force`: `content/_raw/` хранит ручную
вычитку, и больше нигде её нет.

## Жёсткие правила (рельсы — нарушать нельзя)
- **Две среды:** `(vitrina)` — со скроллом и моушеном; `(chitalnya)` — БЕЗ Lenis и
  тяжёлого моушена (длинный текст несовместим с инерционным скроллом).
- **reduced-motion — паритет:** у каждого эффекта полноценный статический сценарий,
  а не «выключено».
- **Доступность — канон:** основная навигация по «Колесу» — доступный `<WheelIndex />`
  (список «тема → рассказы»); визуальный граф — улучшение поверх. Индекс раньше графа.
- **Кириллица обязательна** во всех шрифтах; проверять рендер «», тире, ударений (во́ду).
- **Движется только «сверхъестественное»**; контент стабилен. Никакого тотального parallax.
- Контент статичен → SSG; тяжёлые острова через `next/dynamic` (`ssr:false` для WebGL и графа).
- НЕ использовать Contentlayer (заброшен). Lenis ИЛИ ScrollSmoother — не оба.
- WCAG AA по контрасту. Никаких >3 вспышек/сек.

## Что значит «готово»
Слово «готово» произносится только с доказательством. Обязательный минимум:

1. **`/verify` прогнан** — типы, линт, тесты, сборка. В отчёте статус каждого шага.
2. **Визуальные правки** — `/verify-in-browser`, снимки приложены **до** слова «готово»,
   а не после жалобы владельца. Минимум: десктоп, мобильная ширина, reduced-motion.
3. **Запись в `ИЗМЕНЕНИЯ.md`** (`/changelog`).
4. **Финальный отчёт** с разделом «за вами» — что осталось проверить человеку.

Чего делать нельзя: писать «должно работать», «скорее всего», «обновил документацию»
без указания файлов. Не проверил — так и скажи прямо.

## Процесс
- **Одна задача — одна сессия.** Контекст между сессиями держат `CLAUDE.md`, `docs/` и
  git-история; чаты-долгожители деградируют (компакции теряют детали).
- **Никогда не переноси фидбэк владельца в трекер, не сверив с кодом.** Сырая жалоба из
  браузера — не задача, пока не проверено, что дефект жив. Именно так в июне 2026
  четыре уже исправленных бага месяц числились открытыми.
- Один логический кусок — один коммит с понятным русским сообщением.
  `git push` — только по явной просьбе.
- **Рискованные правки — сначала короткий план и подтверждение:** граф «Колеса»,
  контент-пайплайн, глобальные токены/темы, всё наружное (выгрузки на сторонние
  сервисы, деплой).
- Вес страниц вырос осознанно — подними потолок в `scripts/check-bundle-size.mjs` тем же
  коммитом и объясни почему. Снизил вес — опусти потолок тем же коммитом.
- Писать минимально необходимое, без лишних абстракций; ключевые решения объяснять
  коротко в комментарии — «почему», а не «что».

## Windows-грабли (проверено на опыте — не переоткрывать)
- **Python + кириллица**: без UTF-8 падает на cp1251. `PYTHONUTF8=1` уже задан для
  сессий Claude Code (`.claude/settings.json` → env); в чужом терминале — вручную.
- **Порт 3000 занят** призрачным dev-сервером прошлой сессии:
  `Get-NetTCPConnection -LocalPort 3000 -State Listen | % { Stop-Process -Id $_.OwningProcess -Force }`.
- **Паника Turbopack** «Failed to restore task data (corrupted database)» — удалить
  `.next` и перезапустить. Она же ловится после удаления маршрута: `.next/types` хранит
  ссылку на несуществующую страницу.
- **Кириллический путь `D:\Колесо`** искажается в консольных трейсбеках и ломает наивные
  ESM-URL: node-скрипты запускать из корня проекта; пути в ESM строить через
  `new URL(..., import.meta.url)` / `pathToFileURL`.
- **Поиск по репозиторию** — исключать `.next/`, `node_modules/`, `ds-bundle/`
  (иначе таймауты ripgrep и шум). Использовать инструмент Grep, а не `grep` через Bash.

## Параллельные под-агенты — владелец разрешил и просит
Если задача раскладывается на независимые части (разведка по разным частям кода, анализ
нескольких файлов или логов, сбор фактов из разных источников, несколько независимых
проверок) — запускай под-агентов параллельно, не дожидаясь отдельной просьбы: это явное
постоянное разрешение владельца. Условие одно — качество финального результата не должно
падать: связанные правки, требующие единого видения, делай сам последовательно.
Независимые вызовы инструментов группируй в один блок; долгие команды — в фон.
