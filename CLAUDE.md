# Сайт Евгения Кирилова — «Боковым зрением»

## Что это
Персональный авторский сайт писателя Евгения Кирилова.
Статический контент-сайт (Next.js App Router, SSG/ISR) с «островами» тяжёлой
клиентской интерактивности. Бэкенд и БД для канона не нужны.
Концепция (v2.0): «офисная готика» (жуткое и чудесное внутри обыденного,
«одержимый» интерфейс) · «Колесо» — живая карта смыслов как дом и навигация ·
типографика-герой (дуальная система «душа против системы»). Метафора сайта —
чудо видно «боковым зрением».

**Статус: сайт построен** — пошаговый план (§14 тех-спецификации, фазы 0–6)
выполнен; проект в режиме доработок без фиксированного плана.
Источник истины по смыслу и стилю — `/docs/концепция.md`: при противоречии или
пробеле — остановиться и спросить, не отклоняться молча.
`/docs/тех-спецификация.md` — справочник «как устроено» и почему (архитектурные
решения остаются в силе, пошаговый план — уже история).

## Стек (пинить актуальный stable; сверять `npm view <pkg> version`)
Next.js 16 (App Router) · React 19 · TypeScript 5 · Tailwind v4 (CSS-first, OKLCH) ·
Motion (motion/react) · GSAP 3.13+ · Lenis · d3-force · R3F v9 (точечно) ·
MDX + gray-matter + zod · next/font (cyrillic+latin). Node 20+.
Не выдумывать API: не уверен — проверить (`npm view`) или спросить.

## Команды
```bash
npm run dev        # dev-сервер, http://localhost:3000
npm run build      # прод-сборка (SSG); заодно валидирует контент: невалидный
                   # frontmatter ломает сборку с указанием файла и поля
npx tsc --noEmit   # проверка типов (отдельного скрипта нет)
npx eslint app components lib content   # линт живого кода — НЕ `npm run lint`:
                   # тот захватывает сгенерированный ds-bundle/ (~1850 проблем шума)
```
Тестового фреймворка нет. Проверка изменений — навык `/verify` (типы + линт +
сборка), для визуальных правок — `/verify-in-browser` (Playwright-скриншоты);
финальная проверка глазами — пользователь по чек-листу `docs/проверка-в-браузере.md`.

Контент-пайплайн (Python 3, `pip install pymupdf pillow`; нужен только при
перегенерации контента — на сборке сайта Python не требуется):
```bash
python scripts/ingest_pdf.py              # source/*.pdf → content/_raw/<book>/*.md
python scripts/build_mdx.py               # _raw → content/stories/*.mdx + плашки WebP
python scripts/illustrations_manifest.py  # размеры плашек → content/illustrations.json
```
Артефакты пайплайна детерминированы и коммитятся.

## Архитектура
Алиас импорта: `@/*` → корень репозитория.

**Две среды — две route-группы в `app/`** (реализация первого жёсткого правила):
- `(vitrina)` — `/`, `/workshop`, `/contacts`. Layout оборачивает в `SmoothScroll`
  (Lenis) и слой «офисной готики»: `GrainOverlay`, `CustomCursor`, `ZachinProvider`
  (событие-зачин при входе из «Колеса»). GSAP-скролл-сцены.
- `(chitalnya)` — `/read`, `/read/[book]/[story]`. Нативный скролл; `.reading-root` —
  носитель тем чтения (светлая/сепия/тёмная), тему до отрисовки ставит пре-пейнт
  скрипт в корневом `app/layout.tsx` (анти-FOUC). Ключи localStorage:
  `kirilov:reading-theme`, `kirilov:reading-mode`, `kirilov:reduce-effects`.

**Контент течёт в одну сторону:**
`source/*.pdf` →(ingest_pdf.py)→ `content/_raw/` →(вычитка человеком)→
(build_mdx.py)→ `content/stories/*.mdx` + `public/illustrations/` → загрузчики → SSG.
- `content/schema.ts` — все zod-схемы и канонические ID (`THEME_IDS`, `BOOK_IDS`);
  единственный источник типов контента.
- `lib/content.ts` — загрузка рассказов/книг (fs + gray-matter, zod на сборке).
- `lib/workshop.ts` — отдельный поток «Мастерской» (`content/workshop/*.md`),
  намеренно изолирован как единственный «шов» под будущую CMS. Картинки записей —
  в frontmatter; `content/illustrations.json` — только канонические плашки из PDF.
- `lib/typograf.ts` — русская микротипографика применяется при SSG-рендере,
  не в рантайме.

**«Колесо»:** `lib/graph.ts` — чистые функции (`buildGraph`/`groupByTheme`/
`themeDegree`/`reweight`) над Story+Theme. Потребители: доступный `WheelIndex`
(канон навигации) и визуальный `WheelGraph` (d3-force + SVG, через next/dynamic
ssr:false); раскладка — `lib/wheelLayout.ts`.

**Движение — единый шлюз:** `components/motion/MotionProvider.tsx` считает
`reduced = ОС prefers-reduced-motion ИЛИ ручной тумблер` (localStorage
`kirilov:reduce-effects`); каждый эффект обязан читать его через
`useReducedMotionSafe`. Атмосферные эффекты дополнительно gated по устройству
(`components/haunted/useHauntedCapability.ts`: тач/слабое железо → статичная
деградация).

**Дизайн-токены:** `app/globals.css` — Tailwind v4 CSS-first (`@theme inline`
поверх shadcn-переменных). Семантические цвета готики: `--sodium` (натриевый
свет-«магия»), `--paper`, `--concrete`, `--monitor` — всё в OKLCH. Дуальная
типографика — `lib/fonts.ts`: Source Serif 4 («душа», проза) против Inter +
JetBrains Mono («система», UI); сабсеты cyrillic+latin обязательны.

**Перф-конфиг:** `next.config.ts` — React Compiler включён (автомемоизация
островов), AVIF/WebP, суженные deviceSizes под реальные размеры плашек.

**Не runtime-код:** `ds-bundle/` — сгенерированный экспорт дизайн-системы для
claude.ai/design; `.ds-sync/` — пайплайн этого экспорта (`resync.mjs`:
build → remote-diff → validate → capture; локальный валидатор
`package-validate.mjs` уже встроен в цепочку); `.design-sync/` — entry/previews
бандла; `content/_raw/` — промежуточный этап контент-пайплайна.

**docs/** сверх концепции и тех-спецификации: `отложенное.md` (трекер отложенных
и ручных задач), `definition-of-done.md` (статус §13), `проверка-в-браузере.md`
(ручной чек-лист), `перформанс.md` (метод и бюджеты CWV),
`proofreading-checklist.md`. Трекеры могут отставать от кода — опираясь на них,
сверяйся с фактическим состоянием репозитория.

## Жёсткие правила (рельсы — нарушать нельзя)
- Две среды: (vitrina) — со скроллом и моушеном; (chitalnya) — БЕЗ Lenis и тяжёлого
  моушена (длинный текст несовместим с инерционным скроллом).
- reduced-motion — паритет: у каждого эффекта есть полноценный статический
  сценарий, а не «выключено».
- Доступность — канон: основная навигация по «Колесу» — доступный <WheelIndex />
  (список «тема → рассказы»); визуальный граф — улучшение поверх. Индекс — раньше графа.
- Кириллица обязательна во всех шрифтах; проверять рендер «», тире, ударений (во́ду).
- Движется только «сверхъестественное»; контент стабилен. Никакого тотального parallax.
- Контент статичен → SSG/ISR; тяжёлые острова через next/dynamic (ssr:false для
  WebGL и графа).
- НЕ использовать Contentlayer (заброшен). Lenis ИЛИ ScrollSmoother — не оба.
- WCAG AA по контрасту. Никаких >3 вспышек/сек.

## Windows-грабли (проверено на опыте — не переоткрывать)
- **Python + кириллица**: без UTF-8 падает на cp1251. `PYTHONUTF8=1` уже задан
  для сессий Claude Code (`.claude/settings.json` → env); в чужом терминале —
  выставить вручную.
- **Порт 3000 занят** призрачным dev-сервером прошлой сессии:
  `Get-NetTCPConnection -LocalPort 3000 -State Listen | % { Stop-Process -Id $_.OwningProcess -Force }`.
- **Паника Turbopack** «Failed to restore task data (corrupted database)» —
  удалить `.next` и перезапустить.
- **Кириллический путь `D:\Колесо`** искажается в консольных трейсбеках и ломает
  наивные ESM-URL: node-скрипты запускать из корня проекта; пути в ESM строить
  через `new URL(..., import.meta.url)` / `pathToFileURL`.
- **Поиск по репозиторию** — всегда исключать `.next/`, `node_modules/`,
  `ds-bundle/` (иначе таймауты ripgrep и шум).

## Параллельные под-агенты — владелец разрешил и просит
Если задача раскладывается на независимые части (разведка/поиск по разным частям
кода, анализ нескольких файлов или логов, сбор фактов из разных источников,
несколько независимых проверок) — запускай под-агентов параллельно (Agent:
Explore / general-purpose), не дожидаясь отдельной просьбы: это явное
постоянное разрешение владельца проекта. Единственное условие — качество
финального результата не должно падать: связанные правки, требующие единого
видения, делай сам последовательно. Независимые вызовы инструментов группируй
в один блок; долгие команды (сборка, dev-сервер) — в фон.

## Процесс
- Одна задача — одна сессия. Контекст между сессиями держат CLAUDE.md, docs/ и
  git-история; чаты-долгожители деградируют (компакции теряют детали).
- Один логический кусок — один коммит с понятным русским сообщением.
  `git push` — только по явной просьбе.
- Рискованные правки (граф «Колеса», контент-пайплайн, глобальные токены/темы) —
  сначала короткий план, подтверждение, потом код.
- Версии — актуальный stable, сверять перед установкой. Писать минимально
  необходимое, без лишних абстракций; ключевые решения объяснять коротко.
- Перед «готово» — `/verify`; визуальное — `/verify-in-browser`; запись в
  `ИЗМЕНЕНИЯ.md` (`/changelog`); финальный отчёт с разделом «за вами —
  live-проверка» (чек-лист §13 / `проверка-в-браузере.md`).
